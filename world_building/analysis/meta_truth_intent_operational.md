# What is Truth? What is Intent? (Operational)

This box uses **operational truth**, not metaphysical truth.

## Truth (in the box)
Truth = a claim that is safe to act on **because it is anchored**.

- A claim is *anchored* when it has:
  - a source file path
  - a snippet (or datum)
  - a stable hash of that snippet/datum
  - a claim_id that can be referenced by other artifacts

So "truth" here is not “cosmic certainty” — it's **actionable reliability under audit**.

### ημ mapping
- η = what other agents can verify: anchors, hashes, logs
- μ = what an agent does: executions, edits, edges
Truth is when μ remains answerable to η.

## Intent (in the box)
Intent = a directional potential that predicts or constrains action.

We model intent as:
- typed events (VOW / COMMAND / CHOICE / PLAN / REFUSAL / THREAT …)
- scalar intensity (0..255)
- polarity (+/-)
- circuit salience (which of the 8 circuits it energizes)

Intent becomes *real* when it changes μ:
- it generates forks (Forkgate)
- it changes policy (Permissions/Pack)
- it produces irreversible commitments (Protocol/Law)

## Can intent be executable from a disc?
Yes—if you treat intent as **a program with constraints and proofs**:
- intent_program.json (the "do")
- constraints (the "must not")
- anchors (the "why allowed")
- an interpreter/verifier (the "refuse unanchored ops")

That’s "transmission of intent" as **portable execution semantics**, not mind-meld.
The receiver still has autonomy; the interpreter enforces boundaries.
