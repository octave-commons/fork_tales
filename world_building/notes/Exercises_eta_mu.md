# Exercises — ημ / Anchors / Forkgating

These are *practice drills* for “anti pretty graphs” and “anti forks everywhere.”

## Drill 1 — S3 Anchor Stamp
Pick any non-trivial claim you want to make about the story world.
Write:

- claim:
- about:
- snippet (≤ 280 chars):
- why this snippet supports the claim:
- what would falsify it:

**Rule:** if you can’t fill this out, the claim is not allowed to enter the graph.

## Drill 2 — Intent Gate
Write three possible forks for a scene.
For each fork, score:

- intent strength (0–255):
- what vow/refusal/choice powers it:
- which anchor proves that vow exists:
- why this fork matters more than the other two:

**Rule:** Only forks over threshold survive.

## Drill 3 — S5 Resolution Audit
Take a character with two names (handle + avatar + in-world).
Write an S5 change record:

- entity_id:
- proposed alias / merge / split:
- rationale:
- witness:
- anchor:

**Rule:** alias changes are claims; claims need anchors.

## Drill 4 — Song Compression
Take a fork rationale and compress it into:
- one line (anchor refrain)
- four lines (verse)
- eight lines (mini-song)

**Rule:** compression without distortion. Keep evidence intact.
